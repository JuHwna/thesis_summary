# AI 모델 vs AI 코딩 도구
- 모델과 도구의 차이 + Claude Code 열광 이유

## AI 모델이란
- 언어를 이해하고 답변을 생성하는 AI
- 수십억 개 텍스트 데이터 학습 -> 언어 패턴과 맥락 이해
- ChatGPT, claude.ai에서 질문하면 답변해주는 것

## AI 모델 한계
- 대화만 가능, 실제 작업은 불가
- 할 수 있는 것 : 생각, 판단, 대답
- 할 수 없는 것 : 파일 생성, 코드 생성, 폴더 정리
  - 방법만 알려줌

## AI 모델 종류
- 회사마다 훈련 철학이 다름
- 어떤 데이터, 어디에 중점 => 다른 답변 스타일
  - Claude : 신중하게
  - GPT : 폭넓게
  - Gemini : 멀티미디어
- AI 모델 ①
  - Claude - Anthropic
  - 원칙 기반 훈련 : AI에게 원칙을 가르쳐서 스스로 판단
  - 코딩 정확도 높음, 신중한 답변
- AI 모델 ②
  - GPT - OpenAI
  - 피드백 기반 훈련 : 사람들 선호 답변을 대량 학습
  - 범용적, 유연한 답변에 강함
- AI 모델 ③
  - Gemini - Google
  - 멀티모달 설계 : 텍스트, 이미지, 영상 동시 이해
  - 멀티미디어 이해에 특화

- AI 모델 정리
  - AI 모델 = 두뇌(생각O, 행동X)
    - 회사마다 훈련 방식이 달라서 같은 질문에도 다른 스타일로 답변
- AI 코딩 도구 : 실제로 행동하게 만듦

## AI 코딩 도구란
- AI모델에 실제 행동 능력을 부여
- AI 모델 = 두뇌
- AI 코딩 도구 = 손과 발

## 차이점
- 모델은 알려주고 도구는 실행한다
  - AI 모델만 : 이 버그는 이렇게 고치면 돼
  - AI 도구 : 파일 생성, 코드 실행, 결과 확인

## AI 코딩 도구 종류
- 크게 3가지 유형
  - 터미널, 데스크톱 IDE, 웹 IDE
- 유형 ①
  - 터미널 기반 - 자율적 작업
    - 목표만 말하면 프로젝트 전체 분석 -> 자율 작업
  - 대표 : Claude Code, Gemini CLI
  - 만들어줘 -> 파일 생성 -> 코드 작성

- 유형 ②
  - 데스크톱 IDE - 코드 편집기 내 지원
  - 코드 작성 중 AI가 다음 코드 예측 -> 자동 완성
  - 대표 : Cursor, Github Copilot, Windsurf
- 유형 ③
  - 웹 IDE - 설치 없이 브라우저에서
  - 브라우저만 열면 바로 코딩 시작
  - 대표 : Replit Agent, Bolt, Lovable
  - 초보자/프로토타이핑에 적합, 복잡한 프로젝트는 한계

- 선택 가이드
  - 나에게 맞는 도구는?
    - 빠른 시작 : 웹 IDE, 진입장벽 ↓
    - 코드 직접 수정 : 데스크톱 IDE, 일상 개발
    - 프로젝트 전체 : 터미널, 가장 강력

## 중요 포인트
- 같은 Claude 모델, 다른 경험
  - 여러 도구에서 같은 모델 사용 -> 모델은 같지만 경험은 완전히 다름
  - ex) 클로드 코드 -> 클로드, 커서 -> 클로드
    - 같은 모델이지만 최종 결과, 경험은 다름

- 도구별 차이
  - 각 도구가 AI를 활용하는 방식이 다름
  - 시스템 프롬프트
    - 기본 지시가 다름
  - 에이전트 루프
    - 일하는 방식이 다름
  - 컨텍스트 관리
    - 기억하는 방식이 다름
  - 전용 기능
    - Skills, Memory, 서브에이전트 등
- 비유
  - AI 모델 = 엔진, AI도구 = 차종

- 트렌드
  - 코드 편집기 기반 vs 터미널 기반
  - 방식 : AI가 코드를 이해하는 범위가 근본적으로 다름
  - 코드 편집기 기반
    - 열린 파일 중심 -> 빠른 응답
      - 내비게이션처럼 직접 운전하면서 막히는 곳에서만 도움
    - 대표 : Cursor, Windsurf, 안티그래비티
  - 터미널 기반
    - 프로젝트 전체 분석 -> 자율 작업
      - 자율주행처럼 목적지만 말하면 AI가 알아서 데려다줌
    - 대표 : Claude Code, Codex CLI, Gemini CLI
   
- ai 모델 성능 비약적 향상
  - AI가 생성하는 코드 품질 올라감 -> 직접 타이핑할 게 줄어듬

- Claude Code
  - 터미널 기반 자율주행의 대표 주자
  - 개발자 : 무엇을 만들지 명령
  - AI : 작업 수행 -> 결과 제출
  - 코딩 역할 : 타이핑 -> 의사결정
 
- Claude Code 차별점
  - 터미널 도구 중에서 뭐가 다를까?
  - 1) 컨텍스트 엔지니어링
    - 프롬프트 -> 컨텍스트 엔지니어링
    - 예전 : 어떻게 말할지가 중요
      - 마법의 문구를 찾는게 핵심 : 단계별로 생각해줘, 전문가처럼 답변해줘.
    - 지금 : 무엇을 알고 있는지가 중요
      - AI가 무엇을 알고 있는지가 중요
    - Shopify CEO(2025.06) : AI 시대 핵심 역량은 컨텍스트 엔지니어링
      - LLM이 작업을 해결할 수 있도록 모든 맥락을 제공하는 기술
    - Langchain 2025 보고서
      - AI 에이전트 실패의 주원인 = 컨텍스트 관리
        - 57% : AI 에이전트 운영
        - 32% : 품질 문제 경험
      - 불필요한 정보 쌓임 -> 초기 지시 묻힘 -> 방향 상실
      - AI 모델 능력 부족 X
    - 컨텍스트 엔지니어링 도구 제공
      - 핵심 : "필요한 정보만 정확하게"
      - 컨텍스트 도구 : Claude Code의 5가지 무기
      - Claude Code의 5가지 무기
        - CLAUDE.MD
          - 프로젝트 규칙 미리 전달(매번 설명X) 
        - Skills
          - 필요할 때만 전문 지식 활성화
        - MCP
          - 토큰 46.9% 절감
          - MCP TOOL 서치를 활용 => 필요한 도구만 찾아서 쓴다.
          - 컨텍스트 윈도우에 토큰을 효과적으로 절감 가능
        - Hooks
          - 상황별 맥락 자동 주입
        - Sub-agents
          - 요약될 결과만 반환 -> 메인 대화 깔끔 유지
    - 정리
      - 많은 정보X-> 필요한 정보를 필요할 때에
      - 이게 컨텍스트 엔지니어링, Claude Code가 이걸 가능하게 함

  - 2) 네이티브 통합
    - Claude Code는 Anthropic이 직접 개발 = Claude를 만든 회사가 만든 도구 
    - 다른 도구 vs Claude Code
      - 다른 도구 : 메뉴얼만 보고 운전
        - 외부 API로 호출하는 방식
        - 공개된 API 문서만 보고 클로드 활용
      - Claude Code : 설계 엔지니어가 운전
        - 클로드가 어떤 프롬프트에 잘 반응하는지
        - 어떤 컨텍스트 구조가 가장 효과적인지
        - 토큰을 어떻게 배치해야 최적의 결과가 나오는지 정확히 알고 있음


## 각 폴더 안에서 모델 세팅하는 방법
- .claude라는 디렉토리를 만들기
  - 그 안에 settings.json 만들기
    - 그 파일 안에 {"model" : "sonnet"}
  - 이러면 클로드 코드 종료한 다음에 다시 켜도 model이 sonnet임

- 구현을 바로 들어가지 않음
  - 탐색이나 계획을 할 때는 plan 모드를 할 수 있음.
  - plan 모드
    - plan 모드는 클로드 코드에게 어떠한 것을 구현하게 하기 전에 계획부터 할 수 있도록 하는 모드
    - shift+tab 두 번정도 누르면 플랜모드 활성화
      - 그 다음에 바로 프롬프트 입력해서 진행
      - 최근에 플랜 모드를 했는데 계획 후 바로 만들어 주는 경우 발생 => 요새 모델은 간단한 거는 바로 만들어주는 경우가 존재.
      - 그래서 프롬프트 끝에 "계획해줘" 이거를 붙여주면 계획만 해주니 이 점은 참고해야함.
    - 이후 만든 계획을 보고 괜찮으면 진행하는데 별로고 수정사항이 있으면 4번으로 가서 ~~~ 수정해줘를 해야함.
    - 이렇게 해야지 토큰 소모가 덜 소모됨
    - 3번 : 울트라플랜이라는 기능으로 더 정교하게 진행할 수 있음.
    - 1번, 2번 : 계획 승인을 하는 것
      - 2번 : 어떠한 파일을 만들 때 클로드가 만든 파일을 매번 확인한다는 것
      - 1번 : 그냥 진행(클로드 코드가 진행할 때 파일 만들 때마다 알아서 진행하라고 한것)
