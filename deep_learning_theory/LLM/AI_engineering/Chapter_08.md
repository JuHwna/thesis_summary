# Chapter 08. 데이터셋 엔지니어링
- 모델의 품질 : 학습 데이터의 품질에 달려 있음.
- 데이터셋 엔지니어링의 목표 : 최고의 모델을 학습할 수 있는 데이터셋을 만드는 것
  - 할당된 예산 내에서 이를 달성하는 것
 
- 데이터 운영
  - 더 이상 여유 시간에 처리하는 부수적인 업무가 아니라 그 자체로 전문적인 전담 역할이 되었음
 
- 데이터 환경에 대한 개요와 자체 데이터셋을 구축할 때 고려해야할 사항들
- 데이터 큐레이션부터 시작해서 어떤 데이터가 필요한지, 얼마나 많이 필요한지, 고품질 데이터란 무엇인지 같은 질문을 다룸
  - 데이터 합성과 처리 기법들을 살펴봄
- 데이터 큐레이션, 생성, 처리는 순차적인 과저을 따르지 않고 여러 단계를 오가며 작업하게 됨

- 같은 모델이라도 학습 단계마다 학습하려는 능력이 다르기 때문에 각각 다른 속성을 가진 데이터셋이 필요함
  - ex) 사전 학습에서는 데이터 양을 보통 토큰 수로 측정
  - 지도 파인튜닝에서는 예시 수로 측정하는 등 세부적인 접근 방식에는 차이가 있음
 

- 데이터 중심 AI의 관점
  - 모델 중심 AI : 모델 자체를 개선해서 AI 성능을 올리는 방식
    - 새로운 아키텍처를 설계하거나 모델을 더 크게 만들거나 새로운 학습 기법을 개발하는 방식
    - 딥러닝 초기에는 AI 벤치마크 대부분이 모델 중심
  - 데이터 중심 AI : 데이터를 개선해서 AI 성능을 올리는 방식
    - 새로운 데이터 처리 기법을 개발하고 고품질 데이터셋을 만들어서 더 적은 자원으로도 더 좋은 모델을 학습시키는 방식
    - 최근에는 데이터 중심 벤치마크가 늘고 있음
      - 같은 모델이 주어지면 이 모델이 최고 성능을 낼 수 있는 데이터셋을 만드는 것
