# Chapter 08. 데이터셋 엔지니어링
- 모델의 품질 : 학습 데이터의 품질에 달려 있음.
- 데이터셋 엔지니어링의 목표 : 최고의 모델을 학습할 수 있는 데이터셋을 만드는 것
  - 할당된 예산 내에서 이를 달성하는 것
 
- 데이터 운영
  - 더 이상 여유 시간에 처리하는 부수적인 업무가 아니라 그 자체로 전문적인 전담 역할이 되었음
 
- 데이터 환경에 대한 개요와 자체 데이터셋을 구축할 때 고려해야할 사항들
- 데이터 큐레이션부터 시작해서 어떤 데이터가 필요한지, 얼마나 많이 필요한지, 고품질 데이터란 무엇인지 같은 질문을 다룸
  - 데이터 합성과 처리 기법들을 살펴봄
- 데이터 큐레이션, 생성, 처리는 순차적인 과저을 따르지 않고 여러 단계를 오가며 작업하게 됨

- 같은 모델이라도 학습 단계마다 학습하려는 능력이 다르기 때문에 각각 다른 속성을 가진 데이터셋이 필요함
  - ex) 사전 학습에서는 데이터 양을 보통 토큰 수로 측정
  - 지도 파인튜닝에서는 예시 수로 측정하는 등 세부적인 접근 방식에는 차이가 있음
 

- 데이터 중심 AI의 관점

## 8.1 데이터 큐레이션
- 데이터 큐레이션 : 모델이 어떻게 학습하는지, 학습에 도움이 되는 자원이 무엇인지 이해하는 분야
- 어떤 데이터가 필요한지는 하려는 일과 모델에게 무엇을 가르치고 싶은지에 달려 있음.
  - 자기 지도 학습 파인튜닝 : 데이터 시퀀스가 필요
  - 지시 파인튜닝 : (지시) 응답 형식의 데이터 필요
  - 선호도 파인튜닝 : (지시, 선호 응답, 비선호 응답) 형식 필요
  - 보상 모델 학습 : 선호도 파인튜닝과 같은 형식을 쓰거나 ((지시,응답), 점수) 형식으로 각 예시에 주석이 달린 점수가 있는 데이터를 사용할 수 있음


- 학습 데이터는 모델이 학습했으면 하는 행동을 먼저 보여줘야 함
  - 생각의 사슬 (CoT)
    - 모델이 단계별 응답을 생성하도록 가르치려면 학습 데이터에 CoT 응답이 들어 있어야 함
   
  - 도구 사용
    - 도구 사용 예시를 다시 보여주면 모델의 도구 사용 능력을 더 확실히 키울 수 있음
    - 도구 사용 데이터를 만들 때는 보통 도메인 전문가를 활용함
    - 시뮬레이션이나 다른 합성 기법을 활용해 도구 사용 데이터를 생성함

- 대화 인터페이스가 있는 애플리케이션용 데이터를 큐레이션할 때는 싱글 턴 데이터가 필요한지, 멀티 턴 데이터가 필요한지, 아니면 둘 다 필요한지 고려해야 함
  - 싱글 턴 데이터 : 모델이 개별 지시에 답하는 법을 가르침
    - 간단해서 구하기 쉬움
  - 멀티 턴 데이터 : 모델에게 작업을 해결하는 방법을 가르침
    - 특별히 만든 시나리오나 더 복잡한 상호작용이 필요한 경우가 많음

- 데이터 큐레이션은 모델이 새로운 행동을 배우도록 돕는 새 데이터를 만드는 것뿐만 아니라 나쁜 행동을 잊게 하려고 기존 데이터를 없애는 것도 포함함

- 데이터 큐레이션의 세 가지 기준
  - 데이터 품질
    - 재료의 품질
  - 데이터 커버리지
    - 적절한 재료 조합을 갖는 것
  - 데이터 양
    - 재료를 얼마나 준비해야 하는지
   
### Chapter 8.1.1 데이터 품질
- 적은 양의 고품질 데이터가 많은 양의 노이즈가 있는 데이터보다 좋은 성능을 낼 수 있음
  - 노이즈가 있는 데이터란 관련이 없거나 일관성이 없는 데이터를 말함
  - Yi 모델 패밀리 연구팀
    - 신중하게 만든 1만 개의 지시가 수십만 개의 노이즈가 잇는 지시보다 훨씬 낫다
  - LIMA 2023 연구
    - 신중하게 큐레이션된 1000개의 프롬프트와 응답으로 파인튜닝한 650억 파라미터 라마 모델이 사람 주석자 기준으로 43%의 경우에서 GPT-4와 비슷하거나 더 좋은 답을 만든다는 것을 보여줌

- 고품질의 데이터란?
  - 고품질 : 데이터가 자신의 일을 효율적이고 안정적으로 하는 데 도움이 되는 데이터
  - 6가지 특성
    - 관련성 : 학습 예시가 모델이 학습하려는 작업과 관련이 있어야 함
    - 작업 요구사항 부합 : 주석은 작업의 요구사항에 부합해야 함
      - 정확한 또는 올바른 대신 부합이라는 표현을 쓴 이유 : 과제에 따라서는 정확하거나 올바른 응답이 사용자가 원하는 것이 아닐 수도 있기 때문
    - 일관성 : 주석은 예시들끼리, 그리고 주석자들 간에 일관되어야 함
      - 좋은 주석 가이드라인이 꼭 필요함
    - 올바른 형식 : 모든 예시는 모델이 기대하는 형식을 따라야 함
      - 불필요한 형식 토큰은 모델 학습을 방해할 수 있으므로 없애야 함
    - 충분한 고유성 : 데이터에서 고유한 예시를 말함
      - 모델 학습에서 중복은 편향을 만들고 데이터 오염을 일으킬 수 있음
    - 규정 준수 : 데이터는 모든 관련 내부 및 외부 정책(법률과 규정 포함)을 지켜야 함
