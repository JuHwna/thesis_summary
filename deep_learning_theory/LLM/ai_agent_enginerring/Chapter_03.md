# Chapter 03. 에이전트 시스템을 위한 UX 디자인
- 사용자 경험(UX) : 성공을 좌우하는 핵심 요소
- 에이전트의 효과성, 신뢰성, 도입을 결정하는 건 => 사용자가 에이전트와 상호작용하는 방식
  - 파운데이션 모델과 에이전트 아키텍처가 뛰어난 기술적 능력을 구현함

- 에이전트 모달리티 유형 비교

|모달리티|보편성|예시 활용 사례|적합한 상황|
|-------|------|-------------|-------------|
|텍스트|매우 흔함|고객 지원 챗봇, 생산성 어시스턴트|명확하고 비동기적이며 검색 가능한 소통이 필요할 때|
|그래픽 사용자 인터페이스(GUI)|흔함|워크플로 오케스트레이션 대시보드, 커서 같은 ai코딩 어시스턴트|시각적 구조, 컨텍스트 관리, 다단계 워크플로가 중요한 경우|
|음성|덜 흔함|시리, 스마트 홈 어시스턴트(알렉사, 구글 홈), 콜센터 자동화|손을 쓰지 않는 상호작용이나 자연스러운 대화가 필요한 경우|
|비디오|드물|가상 튜터, 의료 상담 아바타, 대화형 학습 에이전트|시각적 시연, 풍부한 표현, 몰입형 학습이 필요한 경우|

- 핵심 UX 고려 사항
  - 시간이 길어질 경우 컨텍스트를 어떻게 관리하느냐임
  - 에이전트의 능력, 한계, 불확실성을 명확히 알리는 일은 현실적인 기대치를 설정하고 오해를 방지하는 데 필수
- 신뢰와 투명성은 에이전트 시스템의 긍정적 사용자 경험을 떠받치는 토대


## 3.1 모달리티
- 텍스트 기반 인터페이스 : 명료성, 추적성이 뛰어남
- 그래픽 인터페이스 : 풍부한 시각 정보와 직관적 제어 제공
- 음성 인터페이스 : 손을 쓰지 않아도 편한 편의성을 제공함
- 비디오 인터페이스 : 역동적이고 실시간에 가까운 소통을 가능하게 함
- 에이전트 시스템에서 뛰어난 UX를 제공하기 위한 핵심 강점, 과제, 모범 사례를 검토

### 3.1.1 텍스트 기반 인터페이스
- 에이전트 시스템과 상호작용하는 가장 흔하고 범용적인 방식
  - 채택되는 이유 : 단순성, 친숙함, 기존 워크플로에 쉽게 통합되는 점
  - 장점 : 동기식(실시간) 대화와 비동기식 상호작용을 모두 지원
    - 사용자가 나중에 돌아와도 컨텍스트를 잃지 않고 이어갈 수 있음
    - 텍스트 상호작용은 교환 내역이 명확하게 남음
    - 문제가 생겼을 때 투명성, 책임성, 트러블 슈팅이 쉬워짐
    - 정밀성, 추적성, 비동기 소통이 중요한 상황에서 빛을 발함
      - 생산성 도구에서 강력함
    - 구조화된 데이터베이스에서 특정 질문에 답하거나 데이터를 조회하는 지식 검색 시스템에서도 효과적
    - 접근성과 배포 용이성 덕분에 에이전트 UX 설계의 초석이 됨
   
- 사례
  - 텍스트 기반 모달리티
    - 워프 : 자연어 명령 변황, 지능형 자동완성, 컨텍스트 기반 설명을 통합해 개발자 터미널을 재해석함
      - 커맨드라인을 협업 중심의 ai 증강 작업 공간으로 바꿈
      - 자연어 프롬프트로 복잡한 문법이나 플래그를 외우지 않고도 명령을 생성, 실행, 디버깅할 수 있음
    - 클로드 코드, 제미나이 CLI : 자연어 상호작용을 코드 생성, 실행, 파일 조작으로 확장함
      - 커맨드라인 전문지식이 있는 사람만 쓰던 도구가 초보자와 전문가 모두에게 자연어로 시스템과 상호작용한는 접근 가능한 강력한 통로로 바뀌고 있음
      - ai 터미널 : 대화형 파트너로서 사용자 의도를 해석하고 모범사례를 제안, 실시간으로 오류를 디버깅함
  - 슬랙, 팀즈, 왓츠앱 같은 메시징 앱의 채팅 인터페이스

- 핵심 한계
  - 발견 가능성
    - 사용자가 에이전트가 지원하는 능력이나 효과적인 명령 표현 방식을 모르는 경우가 많음
    - 제공 기능을 추측하거나 기억해야 함
      - **행동 유도성**이 부족하면 혼란이 생기고 에이전트 기능이 충분히 활용되지 않으며 요청이 지원 범위를 벗어날 때 사용자의 불편함이 커집
  - 자연어의 모호성
    - 예상치 못한 방식으로 요청 표현 가능(견고한 의도 인식 필요)
    - 응답 길이 제약의 영향
    - 감정적 뉘앙스 한계

- 효과적인 텍스트 기반 에이전트 설계
  - 발견 가능성을 높이는 전략이 필요
    - 온보딩 메시지, 주기적 기능 안내, 대화 중 동적 제안을 통해 지원 기능을 선제적으로 알려야 함
  - 명확성, 컨텍스트 유지, 오류 관리에 대한 세심한 주의 필요
    - 명확성 : 간결하게 소통(과도한 전문 용어와 장황한 설명은 피해야 함)
    - 컨텍스트 유지 : 대화가 길어지면 컨텍스트를 유지 중요
    - 오류 관리 : 문제 상황에도 우아하게 대응 필요
      - 명확한 오류 메시지와 폴백 메커니즘을 제공함

### 3.1.2 그래픽 인터페이스
- 여러 그래픽 요소를 결합해 사용자와 에이전트 시스템이 시각적인 상호작용을 하도록 함
  - 효과적인 곳 : 시각적 명료성, 구조화된 워크플로, 다단계 프로세스가 필요한 작업
  - 장점
    - 정보를 시각적으로 제시해 인지 부하를 줄인다는 점

- 사례
  - 대시보드 기반 ai 도구, 그래픽 채팅 인터페이스, 클릭 가능한 요소를 갖춘 에이전트 기반 생산성 플랫폼
  - 워크플로를 관리하는 에이전트
    - 대시보드로 대기 작업, 완료 단계, 오류 알림을 보여줘 사용자가 시스템 상태를 한눈에 파악하도록 할 수 있음
    - Langsmith, n8n, Arize, AutoGen
    - 최신 AI 지원 IDE 인터페이스 : 커서, 윈드서퍼, 클라인에 존재
      - 자연어 이해를 코딩 워크플로에 직접 통합해 개발자가 질문하고 코드를 생성 및 리팩터링하고 설명이나 성능 최적화를 받는 일을 하나의 간결한 그래픽 인터페이스 안에서 수행하게 함
  - 데이터 시각화, 구조화된 상호작용, 명확한 상태 업데이트가 필수인 사례에 강함
    - 작업 관리 대시보드, AI 에이전트가 구동하는 데이터 분석 도구, 필터와 시각적 미리보기를 갖춘 전자상거래 추천 시스템, 사용자 질문에 맞춰 구조화된 출력을 동적으로 생성하는 ui 시스템, 하이브리드 워크플로
  - 러버블, 커서, 윈드서퍼, 깃허브 코파일럿 같은 도구 => 고품질 GUI 대거 선보임

- 새롭게 떠오르는 분야: 생성형 UI 등장
  - 정적인 대시보드나 사전 설계도니 레이아웃에만 의존 x
  - 사용자 질의에 따라 인터페이스 요소, 데이터 시각화, 구조화된 출력을 동적으로 만듦
  - 자연어의 유연성과 그래픽 레이아웃의 명료성, 발견 가능성을 결합해 에이전트가 필요할 때마다 풍부하고 컨텍스트에 특화된 인터페이스를 만들어 줌
    - ex) 퍼플렉시티 AI : 텍스트 답변뿐 아니라 질문에 맞춘 지식 카드, 참고 목록, 데이터 테이블을 생성함
      - AI 코딩 코파일럿 : 사용자 의도를 기반으로 전체 폼, 설정 파일, UI 구성 요소를 생성함
  - 효과적이고 사용하기 쉬운 생성형 ui를 만들기 위해선 : 신중한 디자인 패턴, 레이아웃 제약, 우선순위 부여 로직이 중요함
  
- 단점
  - 화면의 물리적 공간이 한정적

- 효과적인 그래픽 에이전트 인터페이스 설계
  - 중요한 정보를 노이즈 속에 파묻지 않도록 우선순위를 정해야 함
  - 인터페이스 반응성을 관리
    - 비동기적 작동 시 : 실시간 업데이트와 상태 전환의 부드러움을 기대함
  - 화면 크기에 맞춰 적응해 일관성 유지 필요
  - 자동화와 사용자 통제의 균형
    - 에이전트가 캘린더 변경을 제안할 때 여러 옵션을 버튼으로 제시해 사용자가 최종 결정을 빠르고 명확하게 내리도록 할 수 있음

### 3.1.3 음성 인터페이스
- 사용자가 말을 주된 소통 방식으로 에이전트 시스템과 자연스럽고 손을 쓰지 않고 상호작용하도록 제공함
- 장점
  - 손을 쓰지 않아도 되는 편의성, 자연스러운 상호작용, 즉각적인 컨텍스트 반응성이 중요한 상황
  - 별도의 조작이 필요 없는 작업, 빠른 질의, 액션 중심 워크플로
  - 편의성, 접근성, 유연성 제공
- 단점
  - 조밀하거나 복잡한 정보를 다룰 때
    - 텍스트 인터페이스가 더 빠른 이해와 참조에 유리함
  - 조밀한 정보 소비나 빠른 훑어보기, 나란히 비교가 필요한 복잡한 의사 결정일 때
  - 


- 사례
  - 아마존 알렉사, 애플 시리 같은 가상 비서
  - 고객 지원 음성 봇
  - 시각 장애가 있거나 이동성이 제한된 사용자에게 접근 가능한 대안 제공


- 옛날 음성 인터페이스의 큰 장애물 : 지연 시간
  - 실시간으로 음성을 처리하려면 음성 인식, 의도 인식, 적절한 응답 생성 필요
    - 대화 흐름을 지연이 발생 => 인터페이스가 어색하거나 기계적으로 느껴짐

- 효과적인 음성 에이전트 인터페이스 설계
  - 중단을 우아하게 처리하는 능력..?
    - 유연성 반영 필요
      - 사용자가 명령을 중간에 멈춰도 혼란 없이 받아들이고 입력을 자연스럽게 수정하게 하며 대화를 새로 시작하지 않고 중단했던 지점에서 이어가야 함
  - 도구 사용
    -외부 컨텍스트 참조, 기록 갱신, 실시간 액션 수행 가능

- 인상적인 발전에도 불구하고 음성 인터페이스는 여전히 개척 단계

- 음성 인터페이스를 배포할 때
  - 사람이 음성과 문자를 처리하는 속도의 차이를 이해해야 함
    - 사람의 발화 속도 : 분당 150 ~ 180단어 정도
    - 읽기 속도 : 평균 분당 250 ~ 300단어
    - 훑어 보기 속도 : 500 이상


### 3.1.4 비디오 기반 인터페이스
- 시각, 청각, 때로는 텍스트 요소를 하나의 일관된 경험으로 결합하는 새로운 에이전트 상호작용 양식
- 줌, 마이크로소프트팀즈, 가상 이벤트 같은 플랫폼을 통해 비디오가 디지털 생활 전반에 퍼지면서 에이전트는 이러한 환경에 통합되는 새로운 방식을 찾고 있음
  - 장점
    - 시각적 단서, 음성, 텍스트 오버레이, 애니메이션 등 여러 감각 채널을 결합해 더 풍부하고 표현력 있는 상호작용을 제공한다는 점
    - 인간과 유사한 표정과 제스처를 모사해 소통에 감정적 뉘앙스를 더할 수 있음
    - 인간스러움을 더해 명료성, 감정적 연결, 전반적 효율을 높임
  - 단점
    - 불쾌한 골짜기
    - 기술적, 디자인적 과제
      - 고품질 비디오 상호작용에 상당한 연산 성능과 대역폭 필요
        - 랙이나 화질 저하를 유발해 사용자 경험을 해칠 수 있음
      - 표정, 제스처, 립싱크가 조금만 어긋나도 불쾌한 골짜기 위험이 커져 몰입보다 불편함을 줄 수 있음
    - 시각 데이터를 고유하는 데 대한 불안 => 프라이버시 우려 증폭
 
- 사례
  - 대면 대화를 모사하는 비디오 아바타부터 실시간 비디오 협업 도구에 내장된 에이전트까지 범위가 넓음
  - 원격의료, 교육, 원격 협업, 인터랙티브 엔터테인먼트 같은 산업
 

### 3.1.5 모달리티 결합을 통한 매끄러운 경험
- 여러 모달리티를 하나의 응집된 사용자 여정으로 결합함
- 사용자는 모발리티의 경계를 의식하지 않음
-
