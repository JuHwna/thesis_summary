# 2. 데이터 분석
## 3차시. AI로 데이터 분석 200% 활용법
### 01. LLM을 활용한 스마트한 데이터 분석
- 잘못된 질문 : llm에 무턱대고 이 데이터 분석해줘
  - 분석이 잘못되거나, 할루시네이션으로 이어질 때가 많음
- 올바른 질문 : 데이터 올려놓고
  - 지난 3개월 매출 데이터에서 요일별 패턴을 찾고 주말 매출이 낮은 이유를 3가지로 가설화해줘.
  - **구체적이고 목적이 명확한 질문이어야 함**

### 데이터 분석 3단계 접근법
(1) 컨텍스트 세팅 
- 맥락 설정 : LLM에 충분한 상황 설명을 해줘야 함

~~~
당신은 피자 체인점의 데이터 분석가입니다.
2015년 한 해 동안 1650건의 주문 데이터를 분석해서 2016년 사업 전략을 수립해야 합니다.
목표는 매출 20% 증대입니다.
이 데이터에는 주문일시, 피자 종류, 사이즈, 가격, 카테고리 정보가 포함되어 있습니다.

이해 하셨나요?
~~~

- GPT5 같은 경우 : 사용자가 입력한 데이터를 토대로 출력을 할 때 추가적으로 어시스턴트 기능이 들어와 있음
  - 더 사용자에게 필요한 질문을 자동으로 생성해주는 걸 물어봐줌
- 가설 검증 요청
- 분석 결과를 액션 아이템 도출하고 실행하는 것
  - 분석 결과를 바탕으로 구체적인 행동을 계획하고 가설을 검증하고
  - Build, measure, learn 이 세 가지 사이클을 돌려야 함
  - LLM을 활용해서 액션 아이템 도출까지 가능
    - 가설 검증에서 실험 설계를 하고 실행, 측정, 학습 가능 => 또 다른 가설 검증도 실행 가능

~~~
위 피자 매출 분석 결과를 바탕으로 다음 관점에서 구체적이고 실행 가능한 액션 아이템을 제안해주세요:
1. 즉시 실행 가능한 액션(1~2주 내)
2. 단기 전략(1~3개월)
3. 중장기 전략(6개월~1년)
각 액션마다 예상 효과와 측정 지표도 함께 제시해주세요.
~~~

## 5차시. 바이브코딩, 프로토타입앱 만들기
- IT 기획자 입장 : 기획서는 완벽하게 만들었는데 팀원들이 실제로 어떻게 작동하는지 보여주세요라고 할 때 써먹으면 된다.
  - 시각적으로 프로토타입을 보여주는게 가장 좋음
- 러버블(Lovable)

### 바이브코딩
- 기존 방식 : 기획 -> 와이어프레임 -> 디자인 -> 퍼블리싱 -> 개발 -> 테스트 -> 배포(몆 주~ 몇 달 소요)
- 바이브 코딩 : 아이디어 -> 웹기반 바이브 코딩 서비스에 설명 -> 30분 만에 배포 완료
  - 핵심 : 완벽한 서비스가 아니라 우리가 빠르게 아이디어, MVP, 시장 검증을 하는 미니멀 바이럴 프로덕트 자체를 우리가 만들어내는 것
    - 최소 기능으로 출시하는 제품
- 러버블 : 브라우저에서 바로 코딩하고 배포할 수 있는 클라우드 개발 환경
  - 자연어로 설명만 하면 완전히 웹서비스를 하나 만들어 줌

이 스케치를 바탕으로 실제 작동하는 레스토랑 예약 앱을 만들어주세요.
[서비스 개요]

[화면별 구성 요소]
1. 장소 목록 화면(place list)
-상단 : 지역 선택 드롭다운(강남구 서초동)
-필터 : 이용인원, 날짜 선택
-카테고리 태크 : 내부식당, 음식종류, 가격, 분위기, 예약
- 레스토랑 카드 : 이미지, 이름, 평점, 주소, 예약 가능 시간, 예약하기 버튼

2.상세정보화면
-뒤로가기, 홈, 북마크, 검색 아이콘
-레스토랑명, 평점 표시
-이용 정보(위치, 운영시간 등)
-전화걸기, 위치보기 버튼
-웨이팅 현황 섹션(Eat In, 대기 인원 수 표시)
-웨이팅 등록 버튼

3. 리뷰 화면
-탭 메뉴: 타임 라인, 팔로잉
-필터 태그 : 전체, 중식, 퓨전, 프랜차이즈, 한식
-사용자 프로필, 팔로우 버튼
-리뷰 내용과 평점

4. 예약 확인 화면
-나의 예약 정보
-예약 상세: 파스타야 서초점, 날짜/시간, 인원 수
-예약 취소, 위치확인 버튼

[기능 요구사항]
-지역별 레스토랑 필터링 및 검색
-실시간 예약 가능 시간 확인
-웨이팅 등록 및 현황 확인
-사용자 리뷰 작성 및 조회
-예약 내역 관리(확인, 취소)
-북마크 기능

[디자인 요구사항]
-색상 : 깔끔한 흰색 배경, 포인트 컬러는 주황색 계열
-카드형 레이아웃으로 정보 구분
-모바일 최적화된 터치 친화적 UI
-직관적인 네비게이션과 탭 구조
