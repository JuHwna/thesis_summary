# 1. n8n 고객대응 에이전트 강좌
## 0. 고객 대응 에이전트 시연
### [실습] 고객 문의에 따른 자동 수행 에이전트 사례 살펴보기
#### n8n으로 만드는 고객 대응 에이전트
##### Agentic Workflow 자동화 플랫폼
- n8n : nodemation(Node.js+automation)
  - javascript의 서버를 실행하는 플랫폼과 automation 자동화라는 뜻의 결합어인 nodemation
  - 노드 하나하나를 ㄹ연결해서 자동화 흐름 형태로 진행되는 n8n의 워크플로우 

<img width="1229" height="412" alt="image" src="https://github.com/user-attachments/assets/5d159dd7-241c-42ea-ae14-060cdc12df38" />

- 반복적으로 하고 있는 업무가 있으면 그 부분들을 업무 순서에 맞추어서 자동화로 만드는 플랫폼
- 예시) 고객 문의를 자동으로 회신하고 또 직원에게 알람으로 갈 수 있는 자동화 플랫폼의 workflow 예시
  - n8n에서 고객의 질문을 받았을 때, llm이 처리를 해주게 됨
    - 고객 질문의 문장에서 어떤 문의가 들어왔는지를 처음에는 분류를 해주게 
    - 분류 다음에 우리가 긴급한 내용이다라고 하면 바로 직원들이 확인할 수 있도록 slack으로 알람이 감
    - 긴급한 내용이 아니면 바로 처리 로그로 가서 고객이 물어봤던 질문들을 상세적으로 기록할 수 있는 부분으로 기록해주고 여기에 접수되었습니다까지 완성하는 흐름
   
- 시연 목표
  - [고객] 문의 접수 -> 문의 분류 -> 필요 시 slack 알림(담당 직원) -> 문의 로그 기록(회사) -> 자동 회신

- 고객 문의 자동 회신 및 알람
<img width="1189" height="491" alt="image" src="https://github.com/user-attachments/assets/ca2e15e8-4782-45e4-9ff4-469f77deb222" />

- 노드 연결 및 노드이 돌아가는지 설명
  - (1) 메시지를 받는 노드
    - 고객이 메시지를 이렇게 입력하게 된다라고 했을 때 그 고객 메세지를 받아서 우리 LLM 텍스트 모델이 받아오게 됨
  - (2) LLM 텍스트 모델에서 받아오는 노드
    - 물어봤던 질문들을 그대로 프롬프트를 받아 오게 됨
    - 질문을 다양하게 분류할 수 있는 프롬프트를 미리 입력해두어야 함
      - 카테고리를 어떻게 나눌 것이냐
      - 질문을 그대로 넘겨주는게 아니라 질문 받아온 내용을 정리할 수 있도록 카테고리로 나눔
      - 얼마나 급한지 그 정도도 한 번 나눠봐서 급하다고 하면 슬랙에 알람이 갈 수 있도록 설정
      - 분류 기준에 대한 설정도 설명을 해놓음
        - 설명해줬을 때 이게 어떤 카테고리에 소속되는지 구분해놓음
      - 최종 출력 형태도 넣음
    - 왼쪽은 INPUT, 오른쪽은 OUTPUT
    - Gemini : 무료로 쓸 수 있는 api기능들, 무료로 쓸 수 있는 사용량도 제공해줌
  - (3) 텍스트를 파싱하는 부분
    - 앞의 output 값들이 이번엔 input으로 가져오게 됨
    - output은 조금 더 정리한 형태로 나오게 됨
      - 구분해서 나올 수 있록 처리를 해주는 부분
  - (4) 분류하는 노드
    - input: 방금 가져온 내용들을 체계에 맞게 정리를 해둠
    - 급한 부분에 대한 구분값을 구한 후에 어떤 구분값일 때 슬랙에 알람을 가게 할 것인지를 정하는 단계
  - (5) 구글 시트에 내용 적힘
  - (6) 고객에게는 문의 접수 알림 보이게 됨
